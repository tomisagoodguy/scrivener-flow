---
description: 為現有的元件或函式產生單元測試
---

這個工作流將協助您自動補齊測試案例。

1. 請輸入您想測試的檔案相對路徑。
2. 我將讀取目標檔案內容以理解其邏輯。
3. 在同一目錄下建立擴充名為 `.test.tsx` (或 `.test.ts`) 的檔案。
4. 使用 Jest 與 React Testing Library 撰寫全面的測試案例（包含 AAA 模式）。
5. 驗證所有 Import 路徑是否正確。

---
**測試結構包含：**
- 每個函式/元件的 `describe` 區塊。
- 多個邊界條件測試。
- 若有需要，自動設定 Mock 選項。
