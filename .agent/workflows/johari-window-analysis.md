# Johari Window Analysis Workflow

此工作流用於當使用者提供深度背景資訊或明確要求進行 Johari Window 分析時啟動。目標是擴大「公開區」，減少「盲區」與「隱藏區」，並探索「封閉區」。

## 執行邏輯

請嚴格依照以下三個維度進行回應：

### ▋ 1. 我知道且 AI 不知道的「隱藏區」 (The Known-Unknown to Shared)

**目標**：確認並吸收使用者提供的獨家資訊（Context Integration）。

- **行動**：
  - 總結使用者剛才提供的關鍵資訊、背景知識或業務邏輯。
  - 明確指出這些資訊如何補充了之前的認知缺口。
  - *這是「同步」的過程。*

### ▋ 2. 我不知道且 AI 知道的「盲區」 (The Unknown-Known)

**目標**：利用 AI 的技術廣度指出使用者的盲點（Technical Enlightenment）。

- **行動**：
  - 檢視使用者的想法中是否存在技術瓶頸、安全風險或架構缺陷。
  - 提出使用者可能沒想到的替代方案或最佳實踐（Best Practices）。
  - *這是「互補」的過程。*

### ▋ 3. 你我都不知道的「封閉區」 (The Unknown-Unknown)

**目標**：共同探索未知的潛能（Synergistic Discovery）。

- **行動**：
  - 當「隱藏區」的業務知識結合「盲區」的技術能力後，會產生什麼新的化學反應？
  - 提出前瞻性的建議、自動化機會，或是產品演進的下一步。
  - *這是「進化」的過程。*

## 觸發時機

- 當使用者輸入詳細的 Context 或 Background 時。
- 當使用者使用 `/johari` 指令時。
- 當需要進行深度策略規劃時。
